# 论文整合与规范化计划

## 任务目标
将8个独立章节文件合并为一个完整的论文文档，并进行全面的格式规范化和质量优化。

## 一、标题格式规范化

### 当前状态
- 使用 Markdown 格式：`#` (一级)、`##` (二级)、`###` (三级)
- 部分使用四级 `####`

### 目标格式
采用数字编号体系，最多四级：
- 一级标题：第X章 XXX（如"第一章 绪论"）
- 二级标题：X.X XXX（如"1.1 课题背景和意义"）
- 三级标题：X.X.X XXX（如"1.2.1 Design-to-Code技术研究进展"）
- 四级标题：X.X.X.X XXX（如"5.2.1.1 文本生成设计稿流程"）

### 转换规则
1. `# 第X章 XXX` → 保持为章标题
2. `## X.X XXX` → 二级标题（已有数字编号的保持，无编号的添加）
3. `### X.X.X XXX` → 三级标题
4. `#### XXX` → 四级标题（需添加编号）
5. 创新点部分（如"创新点一"、"创新点二"）保持原有格式，作为特殊标记

## 二、图表题注规范化

### 当前问题
- 部分使用加粗标题：`**图1-1 前端开发工作流程对比图**`
- 部分直接使用文本：`图3-1: 系统功能架构图`
- 表格题注格式不统一

### 规范方案
1. 所有图题注统一格式：`<p align="center">图X-X XXX</p>` 或普通文本行
2. 所有表题注统一格式：`<p align="center">表X-X XXX</p>` 或普通文本行
3. 不使用 Markdown 标题语法（`#`或`**`）
4. 图表编号按章节顺序重新梳理，确保连续性
