#### 2.2 各模块开头段落重写

**3.1.1 设计稿生成模块 (L11)**
```markdown
旧: 设计稿生成是系统的入口功能，用户需要通过自然语言描述或上传参考图片两种方式获取初始设计稿。这一环节直接决定了后续编辑和代码生成的质量基础。

新: 设计稿生成模块作为系统的核心入口，为用户提供文本驱动与图片驱动两种设计稿创建通道。该模块对接后端AI服务能力，将用户的自然语言描述或UI参考图片转化为结构化的Design JSON数据，为后续的可视化编辑与代码生成环节奠定数据基础。
```

**3.1.2 可视化编辑模块 (L21)**
```markdown
旧: 当设计稿生成完成后，用户需要进入可视化编辑器对其进行精细化调整。编辑器采用"左侧画布渲染 + 右侧属性面板"的经典布局范式，画布区域实时呈现Design JSON对应的可视化效果，属性面板则展示当前选中组件的可编辑字段[20]。

新: 可视化编辑模块采用"左侧画布实时渲染 + 右侧属性面板调控"的经典布局范式，为用户提供所见即所得的设计稿精调能力。该模块基于DesignRenderer递归渲染引擎实现Design JSON到可视化DOM的实时映射，配合属性面板的双向绑定机制，支持组件选中高亮、样式属性编辑、拖拽层级调整、撤销重做等五类核心交互操作[20]。
```

**3.1.3 代码生成与导出模块 (L35)**
```markdown
旧: 设计稿经编辑确认后，用户需要将其转换为可在实际项目中运行的前端源代码。代码生成模块基于确认后的Design JSON，通过模板映射引擎生成指定前端框架的目标代码[21]。

新: 代码生成与导出模块承担将确认后的Design JSON转换为可执行前端源码的任务。该模块内置React、Vue、HTML三种目标框架的代码模板引擎，采用适配器模式封装不同框架的生成逻辑，并提供代码预览运行与项目级打包下载的一站式交付能力[21]。
```

**3.1.4 用户管理与历史记录模块 (L45)**
```markdown
旧: 为保证多用户环境下的数据隔离与持续性访问，系统需要建立完善的用户身份认证与会话持久化机制[22]。

新: 用户管理与历史记录模块构建了多租户环境下的身份认证体系与数据持久化层。该模块基于JWT Token实现无状态会话管理，通过MongoDB存储用户信息与操作历史，并以userId+sessionId的复合键实现严格的数据隔离与跨会话工作接续能力[22]。
```
